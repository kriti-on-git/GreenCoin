# Contributing to GreenCoin

Thank you for taking the time to contribute to GreenCoin! Please follow these guidelines to keep our codebase clean, consistent, and maintainable.

---

## 🌿 Branch Naming Convention

We use specific prefixes for branch names to identify the purpose of the work. Use lowercase and separate words with hyphens.

*   `feature/` — For new features or enhancements.
    *   *Examples:* `feature/auth`, `feature/wallet`
*   `fix/` — For bug fixes.
    *   *Example:* `fix/login`
*   `refactor/` — For code changes that neither fix a bug nor add a feature.
    *   *Example:* `refactor/api`
*   `docs/` — For documentation-only changes.
    *   *Example:* `docs/readme`

---

## 💬 Commit Message Style

We follow lightweight conventional commit formats. Write your commit messages in the imperative mood (e.g., "add feature" instead of "added feature").

*   `feat:` — A new feature for the user.
*   `fix:` — A bug fix for the user.
*   `refactor:` — A code change that neither fixes a bug nor adds a feature.
*   `docs:` — Documentation-only changes.
*   `test:` — Adding missing tests or correcting existing tests.
*   `chore:` — Changes to the build process, auxiliary tools, or libraries (e.g., updating dependencies).

*Example:* `feat: add wallet balance auto-refresh`

---

## ✅ Pull Request (PR) Checklist

Before submitting your PR for review, ensure you can check off all of the following items:

- [ ] **Tested locally:** The changes have been verified and work as expected on your local environment.
- [ ] **No console errors:** The application builds and runs cleanly without throwing logs, warnings, or errors in the console.
- [ ] **Updated documentation:** Any code changes affecting setup, APIs, or user workflows are updated in the documentation.
- [ ] **Demo video attached:** A short screen recording or GIF is linked in the PR description to demonstrate visual or behavioral changes.

---

## 🐛 Issue Reporting

If you find a bug or have a feature request, please open an issue using the template below.

### Issue Template

```markdown
### Description
[Clear and concise description of what the bug or request is]

### Expected Behavior
[What you expected to happen]

### Current Behavior
[What actually happens instead]

### Steps to Reproduce
1. Go to '...'
2. Click on '....'
3. Scroll down to '....'
4. See error

### Screenshots
[Attach or paste screenshots/GIFs here if applicable]
```
